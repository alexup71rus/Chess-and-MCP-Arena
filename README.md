# ♛ Шахматы

Классические шахматы со всеми правилами. Два игрока за одним устройством (горячие места).

Написано на **React + TypeScript + Vite**. Движок (генерация ходов и правила) полностью
изолирован от UI и покрыт тестами, включая [perft](https://www.chessprogramming.org/Perft_Results) —
золотой стандарт проверки корректности генерации ходов.

## Возможности

- ✅ Все правила шахмат: ходы 6 фигур, рокировка (со всеми условиями), взятие на проходе,
  превращение пешки
- ✅ Корректное определение шаха, мата, пата
- ✅ Ничьи: правило 50 ходов, троекратное повторение позиции, недостаточный материал
- ✅ История ходов в стандартной алгебраической нотации (SAN) с устранением неоднозначности
  и обозначениями шаха (`+`) и мата (`#`)
- ✅ Взятые фигуры и материальный перевес
- ✅ Подсветка: выбранная фигура, возможные ходы (точки/кольца), последний ход, король под шахом
- ✅ Отмена хода, переворот доски, новая партия
- ✅ Диалог выбора фигуры при превращении пешки
- ✅ Адаптивная вёрстка

## Команды

```bash
pnpm install      # установить зависимости
pnpm dev          # запустить dev-сервер (http://localhost:5173)
pnpm build        # production-сборка (tsc + vite build)
pnpm preview      # предпросмотр production-сборки
pnpm test         # запустить тесты движка (vitest)
pnpm test:watch   # тесты в режиме наблюдения
pnpm typecheck    # проверка типов (tsc --noEmit)
pnpm lint         # ESLint
pnpm format       # Prettier (форматирование)
pnpm verify       # всё сразу: типы + линтер + формат + тесты
```

## Архитектура

Проект разделён на два слоя:

```
src/
├── engine/                # Чистый TypeScript, без зависимостей от React
│   ├── types.ts           # Типы: Piece, Position, Move, GameStatus, ...
│   ├── geometry.ts        # Координаты доски, алгебраическая нотация
│   ├── fen.ts             # Парсинг/сериализация FEN, стартовая позиция
│   ├── attack.ts          # isSquareAttacked, findKing
│   ├── moveGeneration/    # Генерация ходов по типам фигур + рокировка
│   ├── legalMoves.ts      # Фильтр: нельзя оставлять короля под шахом
│   ├── makeMove.ts        # Применение хода (иммутабельно)
│   ├── gameState.ts       # Мат/пат/ничьи, ключ позиции для повторений
│   ├── notation.ts        # SAN
│   └── index.ts           # Публичный API движка
└── ui/                    # React-интерфейс
    ├── hooks/useChessGame.ts   # Состояние партии через useReducer
    └── components/             # Board, PromotionDialog, MoveHistory, ...
```

Движок принимает решение в виде чистых функций над иммутабельным `Position` —
это делает его простым для тестирования и переиспользования.

## Тесты

Тесты движка (`tests/engine/`) включают:

- **perft** — проверка числа легальных узлов на стандартных позициях (стартовая, Kiwipete и др.)
  на разных глубинах с эталонными значениями. Гарантирует корректность генерации ходов.
- **rules** — мат, пат, ничьи (50 ходов, троекратное повторение, недостаточный материал).
- **special** — рокировка, en passant, превращение, пины.
- **notation** — SAN, устранение неоднозначности, обозначения шаха/мата.
