# ♛ Шахматы

Локальные шахматы для людей и AI-агентов. Один Vite-процесс держит React UI,
шахматный движок, stateful HTTP MCP и SSE-обновления доски.

## Режимы

- **Локально** — два человека играют за одним экраном.
- **Человек против агента** — человек выбирает цвет и ходит в UI; MCP-агент
  автоматически занимает оставшуюся сторону.
- **Агент против агента** — два MCP-агента закрепляются за разными сторонами,
  пользователь наблюдает матч в UI.

Online-матч всегда один. Его создаёт или заменяет только пользователь через UI.
Комнат, идентификаторов партий и ручного подключения в интерфейсе нет.

## Запуск

```bash
pnpm install
pnpm dev
```

Адрес фиксирован: `http://127.0.0.1:5173`.

- UI: `/`
- MCP: `/mcp`
- состояние текущего матча для UI: `/api/game`
- обновления доски: `/events`
- health check: `/health`

Если `5173` занят, Vite завершится ошибкой. Это исключает ситуацию, когда UI и
MCP-клиенты случайно подключены к разным процессам.

Проект намеренно не имеет production build: текущее состояние партии и MCP-сессии
живут в одном локальном Vite-процессе.

## Как подключается агент

Пользователь сначала создаёт online-матч в браузере. После этого агент:

1. вызывает `join_game({color?})`;
2. перед каждым ходом вызывает `get_state()`;
3. выбирает фигуру через `legal_moves({from})`;
4. делает `make_move({move, promotion?})`;
5. при желании завершает игру через `resign()`.

Сторона хранится в stateful MCP-сессии. Агент не передаёт цвет при ходе и не
может случайно сходить за соперника. Создание, сброс и откат партии агентам не
предоставляются.

В human-vs-agent цвет можно не указывать: `join_game()` занимает единственную
агентскую сторону. В agent-vs-agent первый агент выбирает `w` или `b`, второй
вызывает `join_game()` и получает оставшуюся сторону.

Расширенная инструкция: [chess-play](./.agents/skills/chess-play/SKILL.md).
Конфиги клиентов: [mcp-config-examples.md](./mcp-config-examples.md).

## Проверки

```bash
pnpm verify
```

Проверяются типы, ESLint, Prettier, шахматный движок, ownership сторон,
stateful MCP-сессии, UI API и SSE.

## Структура

```text
src/
├── engine/                 шахматные правила и нотация
├── mcp/
│   ├── engineApi.ts        единственная активная партия и ownership
│   ├── server.ts           пять MCP tools и инструкции агенту
│   └── httpServer.ts       stateful MCP, UI API, health и SSE
└── ui/                     локальная игра и online-наблюдение
```
