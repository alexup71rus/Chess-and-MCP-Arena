# chess-play: шахматы через MCP

В проекте всегда существует не больше одной активной online-партии. Её создаёт
пользователь в UI на `http://127.0.0.1:5173`; агент не создаёт и не сбрасывает
матч. MCP работает на `http://127.0.0.1:5173/mcp` и закрепляет сторону за
текущей MCP-сессией.

Основные правила также приходят через поле `instructions` ответа MCP
`initialize`, поэтому их получают клиенты, которые не обнаруживают repository
skills.

## Перед игрой

Проверьте `GET http://127.0.0.1:5173/health`. Если сервер не отвечает, попросите
пользователя запустить `pnpm dev`. Не запускайте долгоживущий dev server без
явного разрешения.

Пользователь должен сначала выбрать режим и начать матч в браузере. Если tool
сообщает, что активной партии нет, попросите пользователя создать её через UI.

## Инструменты

| Tool                            | Назначение                                            |
| ------------------------------- | ----------------------------------------------------- |
| `join_game({color?})`           | Закрепить за MCP-сессией агентскую сторону.           |
| `get_state()`                   | Увидеть режим, очередь, статус, историю, FEN и доску. |
| `legal_moves({from})`           | Получить легальные ходы выбранной своей фигуры.       |
| `make_move({move, promotion?})` | Сделать ход за закреплённую сторону.                  |
| `resign()`                      | Сдаться за закреплённую сторону.                      |

## Вход в матч

### Человек против агента

Вызовите `join_game()` без цвета. Сервер автоматически закрепит единственную
агентскую сторону и вернёт, играете вы белыми или чёрными.

### Агент против агента

- Первый агент выбирает свободную сторону:
  `join_game({color:"w"})` или `join_game({color:"b"})`.
- Второй агент вызывает `join_game()` и получает оставшуюся сторону.

Нельзя занять сторону человека, уже занятую сторону или сменить цвет в рамках
той же MCP-сессии. После создания пользователем новой партии вызовите
`join_game` снова.

## Обязательный протокол каждого хода

1. Вызовите `get_state()`.
2. Проверьте, что `status.kind === "ongoing"` и сейчас очередь вашей стороны.
3. Оцените позицию, историю, шахи и угрозы.
4. Выберите свою фигуру и вызовите `legal_moves({from:"e2"})`.
5. Выберите один из возвращённых вариантов.
6. Только теперь вызовите `make_move({move:"e4"})`.
7. После хода остановитесь и ждите соперника.

Не вызывайте `make_move` вслепую. Если варианты выбранной фигуры не подходят,
вызовите `legal_moves` для другой своей фигуры.

## Нотация

- UCI: `e2e4`, `g1f3`, `e7e8q`, `e1g1`.
- SAN: `e4`, `Nf3`, `Bxc4`, `O-O`, `e8=Q`, `Qxf7#`.
- Для превращения можно передать `promotion`: `q`, `r`, `b` или `n`.

## Ошибки

- «Активной партии нет» — пользователь ещё не начал online-матч в UI.
- «Сначала займите сторону» — вызовите `join_game`.
- «Сейчас ход соперника» — не ходите, позже повторите `get_state`.
- «Сторона занята» — выберите другую свободную сторону.
- «Нелегальный ход» — снова вызовите `legal_moves` для своей фигуры.
- «Партия окончена» — ждите, пока пользователь начнёт новую.
